Introduction

During my studies, I interned as a Data Analyst, explored product and program management, leading to over 8 years as a Technical Program Manager. This experience strengthened both my technical and leadership skills.

A few years ago, I developed an interest in cloud technology and began earning certifications, starting with AZ-900 Azure Fundamentals and more recently, AWS Certified Solutions Architect Associate. My passion for cloud computing continued to grow, and I recently started studying for AZ-104 Azure Administrator Associate. However, I struggled to retain the concepts and was looking for a more hands-on approach in cloud, which led me to the Cloud Resume Challenge.

I am passionate about working towards a career as a Cloud Solutions Architect, DevOps, and FinOps professional, with a keen interest in delivering Enterprise AI solutions.

Implementation

Chunk 1: Building the Front End

  • Static Assets: I started by creating and customizing the static assets including index.html, styles.css, and scripts.js by leveraging starter templates.
  • S3 Bucket Setup: I set up an S3 bucket, uploaded the static assets, enabled static website hosting, and configured public access.
  • Security Concerns: I used CloudFront with HTTPS for secure connections.
  • Domain and DNS: I chose Cloudflare for domain management, setting up CNAME records and enabling DNSSEC.

Challenges: Initial attempts using Azure Storage Accounts and CDN for hosting failed due to domain mapping issues, leading me to switch to AWS.

Chunk 2: Building the API

  • DynamoDB: Used to store visitor counts, updated through a Lambda function.
  • Monitoring: Integrated CloudWatch, PagerDuty, and Slack for alert notifications.

Challenges: Issues with Azure CosmosDB and Function Apps prompted another switch to AWS.

Chunk 3: Front-End/Back-End Integration

  • JavaScript Updates: Updated to call the Lambda function URL endpoint for visitor count increment.
  • Testing: Used Cypress for automated end-to-end testing.

Challenges: CORS issues were resolved, but unique visitor tracking using the X-Forwarded-For header proved difficult to implement.

Chunk 4: Automation / CI

  • Terraform: Used for Infrastructure as Code, organizing components into separate .tf files.
  • GitHub Actions: Automated CI/CD processes, dynamically injecting environment variables into JavaScript files.
  • GPG Integration: Configured for signing commits to verify authenticity.

Challenges: Attempted OIDC for secure credential management but ultimately used GitHub secrets.

Reflections

Completing the Cloud Resume Challenge was transformative. This journey taught me the importance of persistence, hands-on learning, and discipline. I encountered numerous challenges but tackled them one step at a time. I avoided looking at others’ solutions to fully immerse myself in the learning process. Consistency, willpower, and a problem-solving mindset are key to success.

Thanks for reading. Moving on to the next challenge!
